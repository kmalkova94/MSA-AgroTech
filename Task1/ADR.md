### <a name="_qmphm5d6rvi3"></a>**Решение**
Диаграмма контекста С1 - С1.puml
В диаграмме выделены основные типы пользователей относительно их взаимодействию с функционалом системы. Это позволяет убрать избыточную детализацию на начальном этапе и сфокусироваться на основных функциях системы.
Система мониторинга взаимодействует с аналитическим модулем Цифровой платформы АгроПром X для передачи аналитических данных для долгосрочного хранения.
Предполагается, что на начальном этапе система будет использовать внешний провайдер для отправки уведомлений пользователям (push, sms, email).
Внешние источники данных (датчики, камеры) напрямую подключаются к системе.


### <a name="_bjrr7veeh80c"></a>**Альтернативы**
Альтернативная диаграмма контекста C1 - C1(alt).puml
В альтернативной диаграмме более детально указаны основные акторы системы, что позволяет точнее продумать use cases работы сисемы. На данном этапе от детализации было решено отказаться для избежания перегруженности.

Также в альтернативном решении предложен вариант использования IoT-шлюза для предварительной обработки данных из внешних источников. IoT-шлюз является точкой агрегации разнородных протоколов подключения, благодаря чему системе мониторинга не нужно поддерживать несколько протоколов подключения. С другой стороны, при выходе из строя IoT-шлюза могут потеряться данные со всех датчиков и камер. Также шлюз является дополнительным компонентом для поддержки, что усложняет архитектуру, поэтому на данном этапе было принято решение в пользу прямого подключения датчиков к системе.
